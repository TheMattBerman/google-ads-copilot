# rsa-playbook.md

## Core principle
Good RSAs are built from proven buyer language, not brainstorming in a vacuum.

The best raw material often comes from:
- high-intent queries
- converting modifiers
- repeated pain phrases
- comparison language
- offer-specific language

## RSA writing rules
- reflect the commercial intent of the query cluster
- match the landing page promise
- use real search language when possible
- avoid generic “all things to all people” headlines
- do not collapse mixed intent into one bland RSA set

## Inputs to mine
- top converting search terms
- high-CTR query modifiers
- common pain phrases
- competitor/comparison language
- offer terms: pricing, demo, quote, trial, book

## Output expectation
For each RSA recommendation:
- target intent theme
- key phrases to include
- headlines to test
- description angles to test
- LP message alignment note
